If you want to learn addon development, start here.

This section is written for the first real addon, not for advanced scripting extensions or large production addons.

## Before you begin

<Callout type="warn">
Use the [ItsMyBot GitHub repository](https://github.com/itsme-to/ItsMyBot), not only a marketplace build. You need the source code, the addon folders, and `_Example` to learn how addons are structured.
</Callout>

Before writing addon code, be comfortable with:

- TypeScript classes
- `extends`
- `async` / `await`
- imports / exports
- basic typing
- [class-validator](https://github.com/typestack/class-validator) decorators

If those basics still feel unclear, learn them first:

- [TypeScript documentation](https://www.typescriptlang.org/docs/)
- [class-validator documentation](https://github.com/typestack/class-validator)

## What you are building first

Your first addon should stay intentionally small:

- one addon class
- one config file
- one slash command or one event

That is enough to understand the full addon flow without mixing too many concepts at once.

## How addons work in ItsMyBot

The important rule is simple:

1. you write source files in `src/addons/<AddonName>`
2. you build the project
3. the bot loads the compiled addon from `build/addons/<AddonName>`

For example:

<Files>
  <Folder name="src" defaultOpen>
    <Folder name="addons" defaultOpen>
      <Folder name="MyAddon" defaultOpen>
        <File name="index.ts" />
        <Folder name="interactions">
          <File name="pingCommand.ts" />
        </Folder>
        <Folder name="resources">
          <File name="config.ts" />
          <File name="config.yml" />
        </Folder>
      </Folder>
    </Folder>
  </Folder>
  <Folder name="build">
    <Folder name="addons">
      <Folder name="MyAddon">
        <File name="index.js" />
        <Folder name="interactions">
          <File name="pingCommand.js" />
        </Folder>
        <Folder name="resources">
          <File name="config.js" />
          <File name="config.yml" />
        </Folder>
      </Folder>
    </Folder>
  </Folder>
</Files>

That is why the build step matters so much. The runtime does not read your TypeScript files directly from `src/addons`.
