## createChannel
Create a channel. 

**Can triggers follow-up actions**
<TypeTable
  type={{
    value: {
      description: 'The name of the channel, support placeholders & list for random names',
      type: 'string | string[]',
      required: true
    },
    'channel-type': {
      description: 'The type of the channel',
      type: 'string',
      default: 'text',
      required: false
    },
    parent: {
      description: 'The parent channel ID',
      type: 'string',
      required: false
    },
    description: {
      description: 'The channel description, support placeholders & list for random descriptions',
      type: 'string | string[]',
      required: false
    },
    'permission-overwrites': {
      description: 'The channel permissions',
      type: 'array of objects with id, allow and deny properties',
      required: false
    }
  }}
/>

```yaml title='Example'
actions:
  - id: createChannel
    args:
      value: "new-channel" # Channel name, support placeholders & list for random names
      channel-type: "text" # Channel type
      parent: "123456789012345678" # Parent channel ID
      description: "This is a new channel" # Channel description, support placeholders & list for random descriptions
      permission-overwrites:
        - id: "123456789012345678" # Role or user ID
          allow: ["VIEW_CHANNEL"] # Permissions to allow
          deny: ["SEND_MESSAGES"] # Permissions to deny
  - id: createChannel
    args:
      value: "simple" # Channel name, support placeholders & list for random names
```

## createThread
Create a thread in a channel. If made in a forum channel, it will create a forum post. If made in a text channel, it will create a thread.

**Can triggers follow-up actions**
<TypeTable
  type={{
    value: {
      description: 'The name of the thread, support placeholders & list for random names',
      type: 'string | string[]',
      required: true
    },
    duration: {
      description: 'The thread auto archive duration in seconds',
      type: 'number',
      typeDescription: 'Time in seconds',
      required: false
    },
    private: {
      description: 'Whether the thread is private or not',
      type: 'boolean',
      default: 'false',
      required: false
    },
    tags: {
      description: 'The thread tags ID (Only for forum posts)',
      type: 'string | string[]',
      required: false
    },
    'disable-mentions': {
      description: 'If true mentions in the message will not ping anyone. Only for forum posts',
      type: 'boolean',
      required: false,
      default: "false",
    },
    components: {
      description: 'The message configuration for the thread content, required for forum posts',
      type: 'Component[]',
      typeDescriptionLink: '/docs/itsmybot/configuration/message#message-configuration',
      required: true,
    },
  }}
/>

```yaml title='Example'
actions:
  - id: createThread
    args:
      value: "Hello there !" # Thread name
  - id: createThread
    args:
      value:
        - "Thread 1" # Thread name
        - "Thread 2" # Thread name
      duration: 3600 # Thread auto archive duration in seconds (1 hour)
      private: true # Whether the thread is private or not
      tags: # Thread tags ID, optional (Only for forum posts)
        - "123456789012345678" # Tag ID
        - "234567890123456789" # Tag ID
      components:
        - type: text-display
          content: "This is a forum post" # Required for forum posts
```

## sendRequest
Send a http request. New variables will be added when you trigger an action after this action. With the format `[[data_<object>]]`. If you want a value that is inside an object, you can add the path with underscore `_`. For example, if you want the value of `user.id`, you can use `%data_user_id%`.
***Can triggers follow-up actions**

<TypeTable
  type={{
    value: {
      description: 'The URL to send the request to',
      type: 'string',
      required: true
    },
    method: {
      description: 'The HTTP method to use',
      type: 'string',
      default: 'GET',
      required: false
    },
    headers: {
      description: 'The headers to send with the request',
      type: 'object',
      required: false
    },
    body: {
      description: 'The body to send with the request. If the body is an object, it will be sent as JSON.',
      type: 'object | string',
      required: false
    }
  }}
/>

```yaml title='Example'
- id: sendRequest
  args:
    value: 'https://api.mcstatus.io/v2/status/java/mc.hypixel.net:25565' # URL
    follow-up-actions:
      - id: sendMessage
        args:
          content: "Players [[data_players_online]]/[[data_players_max]] | Version [[data_version_name_clean]]"
```
